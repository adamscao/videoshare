# 一、系统总体描述

本系统是一个**自建的轻量视频分享站**，包含：

1. **视频上传功能**（支持大文件，Web 上传 + 服务器手工上传两种）
2. **HLS 格式的视频播放**（所有视频都以 HLS 形式对外播放）
3. **视频访问控制**（部分视频公开，部分视频需要密码）
4. **后台管理页面**（管理员登录后可查看、管理、导入视频）
5. **上传权限控制**（上传可以是仅管理员，也可以开放给所有访客）

系统面向的是小范围分享场景，由你一人担任管理员角色，访客通过链接访问播放页，不需要注册账号。

---

# 二、角色与权限

系统包含两类主要角色：

### 1. 管理员（Admin）

* 有权登录后台管理页面
* 可以查看所有视频列表
* 可以修改视频信息（标题、描述、密码保护等）
* 可以删除视频
* 可以设置“上传权限模式”（管理员上传 / 访客上传）
* 可以通过后台按钮手动触发**服务器目录扫描入库**
* 可以通过后台上传视频

### 2. 访客（Visitor）

* 可以访问视频播放链接
* 可以观看**公开视频**
* 可以在知道密码的情况下观看**受保护视频**
* 在“上传权限模式”为访客级时：

  * 可以通过前端页面上传视频（免登录、免密码）

访客不具备删除、查看所有视频列表等管理权限。

# 三、技术架构约束（高层约束，非实现细节）

1. **后端服务使用 Go 编写**

   * Go 服务负责：

     * 提供 HTTP API 和页面（管理页、上传页、播放页）
     * 存取视频元数据
     * 控制访问权限（视频密码、上传权限）
     * 扫描服务器目录并导入视频

2. **统一使用 HLS 作为视频播放格式**

   * 所有对外播放的视频均以 **HLS（m3u8 + 分片）**形式提供
   * 原始上传视频（例如 mp4）会被系统转换为 HLS 并存储对应目录
   * 播放页面通过 `<video>` + HLS 播放方案进行播放

3. **前端访问通过 Nginx 反向代理**（未来部署假定）

   * Nginx 对外暴露域名 / HTTPS
   * Nginx 将请求转发给 Go 后端服务
   * 静态 HLS 资源（m3u8 / ts 分片）也通过 Nginx 对外提供（具体配置不在本需求中）

# 四、功能需求

## 4.1 视频上传（Web 上传）

### 4.1.1 上传权限模式

系统有一个全局配置项：**上传权限模式**，由管理员在后台设置：

* **管理员级上传模式**

  * 只有登录管理员才能看到上传界面并上传视频
  * 访客没有上传入口

* **访客级上传模式（默认）**

  * 任意访问者都可以访问上传页面
  * 无需登录、无需密码即可上传视频
  * 上传完成后，系统返回对应视频的播放链接，用户可自行分享

默认模式为：**访客级上传**。

### 4.1.2 上传页面内容

上传页面提供：

* 选择本地视频文件的控件
* 可选填写：

  * 视频标题（不填则默认使用文件名）
  * 视频描述（可选）
* 可选设置“是否为受保护视频”及密码（是否允许访客在上传时设置密码可作为一个可选项，如果不明确，可以先由系统统一设为“默认公开，管理员后续再改”）
* 上传按钮

### 4.1.3 上传流程

1. 用户选择视频文件并提交
2. 后端接收视频文件并保存为**原始视频文件**（临时路径或原始目录）
3. 系统自动对该视频执行：

   * 转换为 HLS（生成 m3u8 文件和分片文件）
   * 获取时长、大小等基本信息（存入元数据）
4. 系统为该视频建立一条数据库记录，包括：

   * 视频唯一 ID
   * 播放短链接标识（slug）
   * 标题、描述
   * HLS 路径
   * 是否需要密码、密码哈希（如有）
   * 上传时间
5. 上传完成后：

   * 返回一个可访问的视频播放链接（如 `/v/abcd12`）
   * 对于管理员，在后台列表中也能看到该视频

## 4.2 手工上传 + 扫描导入（服务器侧）

除了 Web 上传，管理员可以通过在服务器上手工放置视频文件的方式导入视频。

### 4.2.1 手工上传目录

系统约定一个**手工上传目录**，例如：

```
/videos/import/
```

管理员可以用 `scp` / `rsync` / 复制等方式直接将大视频文件放入该目录。

### 4.2.2 后台“处理手工上传视频”按钮

管理页面提供一个按钮，例如：

> 【扫描导入手工上传视频】

点击该按钮后，系统执行：

1. **扫描 `/videos/import/` 目录**，获取其中的文件列表
2. 对每个新文件依次进行：

   * 检查是否已经导入过（避免重复）
   * 将文件移动/复制到系统的视频存储区域
   * 将其转换为 HLS（统一格式）
   * 生成视频封面（可选）
   * 创建数据库记录（与 Web 上传生成的数据结构一致）
   * 为该视频生成默认的：

     * 标题（默认文件名去掉扩展）
     * 描述（空）
     * 访问权限：默认公开，不需要密码（后续管理员可在后台修改）
3. 导入完成后，在后台视频列表中可看到这些新视频

如果某些文件无法解析 / 不支持，系统可在后台显示简要错误信息（例如：导入失败的文件名列表和原因）。

系统不强制要求自动定期扫描，**要求明确的是：在后台可以手动点按钮触发扫描入库**。

## 4.3 视频播放与访问控制

### 4.3.1 播放链接与播放页面

每个视频对应一个唯一的**播放链接**，形式例如：

```
https://example.com/v/abcd12
```

访问该链接时，系统返回一个播放页面，包含：

* 视频标题
* 视频描述
* `<video>` 播放器，使用 HLS 源（m3u8）
* 若该视频需要密码且当前访问者尚未验证通过，则显示密码输入界面

### 4.3.2 视频访问权限

每个视频具有以下属性：

* **是否需要密码保护**（布尔）
* **视频访问密码**（存储为哈希）

访问逻辑：

1. 若视频设置为**无需密码**：

   * 访问者打开播放链接即可直接播放，不做额外验证

2. 若视频设置为**需要密码**：

   * 访问者打开播放链接时：

     * 首先看到密码输入表单（不显示 HLS 播放器）
   * 访问者输入密码并提交：

     * 系统验证密码是否正确
     * 若正确，将当前会话标记为对该视频“已解锁”
     * 然后刷新或跳转回播放页，展示 HLS 播放器
     * 若错误，显示错误信息并拒绝播放

3. 在会话未过期的前提下，同一浏览器无需对同一个视频重复输入密码。

### 4.3.3 HLS 播放要求

* 所有播放都通过 HLS 形式完成：

  * 播放页引用 `.m3u8` 地址
  * 对应分片文件存放在服务器文件系统中
* 不再对外直接暴露原始大视频文件路径（如 mp4），除非后续扩展增加“下载”功能

## 4.4 后台管理页面

后台入口示例：

```
/admin
```

### 4.4.1 管理员登录

* 后台入口需要管理员登录才能访问
* 登录方式：用户名 + 密码
* 输入正确后进入管理主界面

### 4.4.2 视频列表

后台主界面显示视频列表，包含至少：

* 视频 ID / slug
* 标题
* 上传时间
* 视频大小（可选）
* 是否密码保护
* 播放链接（可复制）
* 编辑按钮
* 删除按钮

### 4.4.3 编辑视频信息

管理员可以对单个视频进行编辑：

* 修改标题
* 修改描述
* 修改“是否需要密码”
* 设置或修改视频访问密码
* 可选：为视频设置“展示封面”（若存在）

### 4.4.4 删除视频

管理员可以删除某一视频，删除时需确认：

* 删除数据库中的视频记录
* 删除对应的 HLS 文件（m3u8 和分片）
* 删除相关缩略图等辅助文件（如有）

### 4.4.5 后台上传功能

在后台管理中，管理员也可以进行**Web 上传**（与访客上传本质相同，但界面可能位于后台）：

* 上传成功后，视频自动进入列表
* 可立即在后台编辑相关信息

### 4.4.6 上传权限设置

后台需要一个配置项，用于选择“上传权限模式”：

* 选项 1：**仅管理员可上传**
* 选项 2：**所有访客可上传（默认）**

该配置项影响：

* 访客是否可以看到上传页面
* 非管理员是否能通过 Web 上传接口提交新视频

### 4.4.7 “处理服务器上手工上传视频”按钮

后台提供一个操作按钮，用于**手动触发服务器目录扫描入库**：

* 触发后系统扫描预设的手工上传目录（例如 `/videos/import/`）
* 按照前述逻辑将新文件导入为视频记录，并转换为 HLS
* 处理结果可以在当前会话中简单反馈，例如：

  * “导入成功 3 个文件，跳过 1 个（已存在）”

# 五、数据与状态（简要）

这里只描述必要的数据概念，不涉具体数据库设计和实现细节。

### 5.1 视频实体需要包含的信息（逻辑层面）

每条视频记录应至少包括：

* 唯一 ID
* 播放 slug（用于构造 `/v/{slug}`）
* 标题
* 描述
* HLS 存储路径（相对路径/标识符）
* 视频大小（可选）
* 视频时长（可选）
* 是否需要密码保护
* 密码哈希（如需要）
* 上传方式（可选：访客上传 / 管理员上传 / 手工导入）
* 创建时间 / 修改时间

### 5.2 管理员账号信息

* 用户名
* 密码哈希

# 六、安全相关需求（非性能/扩展）

* 后台管理页面必须登录后才能访问
* 视频访问密码不以明文形式存储
* 所有的视频访问控制逻辑应在 Go 后端进行判断，然后再向外提供 HLS 播放地址
